# .agent/ Folder — AI-Optimized Documentation

This folder contains comprehensive documentation generated by analyzing the codebase, historical user stories, and git history. It serves as the **shared source of truth** for both AI agents and human developers.

## Purpose

Provide context for:

1. **AI Agents** — Autonomous agents read these files to understand codebase patterns before generating code, tests, and documentation
2. **Human Developers** — Developers reference these files when working on complex features to follow established conventions
3. **AI-Assisted Tools** — Claude Code CLI, OpenAI Codex CLI, Cursor, and GitHub Copilot produce better results when given this context

## Contents

| File | Description |
|------|-------------|
| `CONTEXT_INDEX.md` | **Start here** — master overview, architecture summary, quick reference |
| `TECH_STACK.md` | Languages, frameworks, tools, and versions |
| `ARCHITECTURE.md` | System design, component diagrams, data flow |
| `CODING_STANDARDS.md` | Conventions, patterns, examples (**critical for consistency**) |
| `DATABASE_SCHEMA.md` | Tables, relationships, ORM patterns |
| `API_REFERENCE.md` | Endpoints, authentication, request/response formats |
| `COMMON_PATTERNS.md` | How to add features, handle errors, write tests |
| `FEATURES/` | Per-feature deep dives (one file per major feature) |

## For Human Developers

### Before Working on a Feature

1. **Read `CONTEXT_INDEX.md`** (5 min) — get the big picture
2. **Read relevant `FEATURES/*.md`** — understand the existing implementation in your area
3. **Read `CODING_STANDARDS.md`** — know the patterns to follow

### When Using AI Coding Tools (Claude Code / Codex / Cursor / Copilot)

- Include `.agent/` context in your prompts for consistent results
- Use the helper script: `./scripts/ai-with-context.sh "your task"`
- Force a specific CLI: `./scripts/ai-with-context.sh --tool codex "your task"`
- See [DEVELOPERS.md](../DEVELOPERS.md) for detailed prompt structure

### After Completing a Feature

- Update relevant `FEATURES/*.md` with your additions
- Document decisions in `.ado/stories/US-{id}/DECISIONS.md`
- The next developer (human or AI) benefits from your documentation

## For AI Agents

All agents automatically:

1. Check for `.agent/` folder existence via `ICodebaseContextProvider`
2. Load `CONTEXT_INDEX.md` (always)
3. Load relevant `FEATURES/*.md` (based on work item context)
4. Load `CODING_STANDARDS.md` (always)
5. Include loaded context in AI prompts alongside story-specific details

This ensures AI-generated code follows established patterns regardless of which AI model is used.

## Maintenance

### Manual Updates

- Update `FEATURES/*.md` when completing major features
- Update `CODING_STANDARDS.md` if team conventions change
- Keep documentation current as the codebase evolves

### Automatic Refresh

- Run **CodebaseDocumentationAgent** via the dashboard to regenerate
- **Recommended:** Quarterly or after major refactoring
- The agent re-analyzes the entire codebase and updates all documentation files
- Statistics (files analyzed, features documented, etc.) are updated automatically

## Version Control

> **IMPORTANT:** This folder is committed to version control intentionally.

The `.agent/` folder should **NOT** be in `.gitignore` because:

- Developers need these files for reference during local development
- AI agents need them for context when processing work items
- They are part of the codebase documentation, just like `README.md`
- Helper scripts (`scripts/ai-with-context.sh`) depend on them
