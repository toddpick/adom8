## Overview

Enhances the Planning Agent's feasibility analysis by adding a robust mechanism to detect and flag unverified external API assumptions during story triage. This improvement ensures that stories with potential integration challenges are identified early in the development process.

---

## Changes Made

## Key Changes
- Added 'researchNeeded' field to PlanningReadiness model
- Enhanced system prompt to include explicit detection of external API assumptions
- Implemented comprehensive placeholder and assumption detection logic
- Updated triage gate to handle unverified external dependencies
- Added detailed warning mechanism for stories with unresolved API capabilities

## API Documentation

## PlanningReadiness Model
### New Field: `ResearchNeeded`
- **Type:** `IReadOnlyList<string>`
- **Purpose:** Tracks unverified external API assumptions
- **Example:**
```csharp
var readiness = new PlanningReadiness {
    ResearchNeeded = new[] { 
        "GitHub Issues API does not expose Copilot Agent Tab task GUIDs",
        "Azure DevOps API custom field availability unconfirmed"
    }
}
```

### Triage Gate Behavior
- When `ResearchNeeded` is non-empty, `Proceed` is automatically set to `false`
- Adds a warning note to the implementation plan
- Requires manual investigation before story can proceed

## Usage Examples

### Example Scenario
```csharp
// Story with unverified GitHub API assumption
var story = new StoryWorkItem {
    Title = "Link to GitHub Copilot Agent Tab",
    Description = "Integrate task GUID from GitHub Issues"
};

// Planning Agent will flag this for research
var result = planningAgent.AnalyzeStory(story);
assert result.Readiness.Proceed == false;
assert result.Readiness.ResearchNeeded.Contains(
    "GitHub Issues API does not expose task GUID field"
);
```

### Handling Unverified Assumptions
1. Story is moved to "Needs Revision"
2. Detailed comment added to work item
3. Requires manual verification of API capabilities
