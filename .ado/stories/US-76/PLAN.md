## Technical Analysis

### Problem Analysis
The dashboard's Current Work Item section displays a dash (-) instead of the actual autonomy level because the GetCurrentStatus API endpoint's CurrentWorkItemInfo response object lacks an AutonomyLevel property. The dashboard code expects wi.autonomyLevel but this field is never populated by the API response.

### Recommended Approach
Add AutonomyLevel property to CurrentWorkItemInfo class in GetCurrentStatus.cs and populate it from the active story's work item fields. The dashboard already has the UI logic to display wi.autonomyLevel, so only backend changes are needed. Extract autonomy level from the activeStory object which has access to work item field data.
