## Overview

## Story Progress Fix for Skipped Testing Agent

This update addresses an issue with dashboard progress calculation when the Testing agent is skipped in GitHub Copilot workflows. The changes ensure that the dashboard correctly shows 100% progress and COMPLETED status when the Testing agent is handled by the GitHub Copilot Coding Agent.

---

## Changes Made

### Key Modifications

- Updated `calculateStoryProgress()` to count skipped agents as completed in both numerator and denominator
- Modified `getStoryStatus()` to treat skipped agents as completed when determining overall story status
- Enhanced `formatStatusText()` to display a specific message for skipped Testing agents
- Ensured consistent visual treatment for skipped states in both card and detail views
- Added support for distinguishing skipped states from pending or failed states

## API Documentation

### Progress Calculation Methods

#### `calculateStoryProgress(story, agentNames)`
- **Parameters**:
  - `story`: Story object containing agent statuses
  - `agentNames`: Array of agent names to evaluate
- **Returns**: Percentage of completed/skipped agents
- **Behavior**: 
  - Counts both 'completed' and 'skipped' statuses as completed
  - Calculates progress as (completed agents / total agents) * 100

#### `getStoryStatus(story)`
- **Parameters**: `story` object with agent statuses
- **Returns**: Overall story status ('completed', 'in-progress', 'failed')
- **Changes**:
  - Skipped agents now count as completed
  - Prioritizes status: failed → in-progress → completed
